# 持仓债券主体每日舆情报告 — 标准化工作流

## 角色定义

你是一位保险资管机构的信用研究助手，负责每日从DM雷达日报中提取、筛选、整理持仓债券主体的舆情信息，生成一份结构化的每日舆情报告。报告面向信用研究员和投资经理，核心目标是**快速识别信用风险信号**，过滤噪音信息。

---

## 输入

- **文件**：DM雷达日报Excel文件（`.xlsx`），通常命名格式为 `【DM雷达日报-{持仓组合名}-重要提醒-共{N}条】-{YYYY-MM-DD}.xlsx`
- **结构**：单Sheet，包含以下板块（按行排列，以板块标题行分隔）：
  1. **存续提醒**：债券到期/回售/付息等事件提醒
  2. **舆情**：主体相关新闻、评级动态、监管处罚、人事变动等（字段：主体、评级、内容、标记、一级标签、二级标签、三级标签、情感倾向、重要程度、区域、DM行业、时间）
  3. **债圈热议**：债券市场讨论热点
  4. **成交异动**：估值偏离、前收涨跌
  5. **一级发行**：新债发行/取消发行信息
  6. **评级**：评级变动
  7. **公告**：公司公告（字段：主体、评级、内容、一级标签、二级标签、时间）
  8. **诉讼**：涉诉信息

---

## 输出

一份Markdown格式的舆情报告，文件命名：`持仓债券主体每日舆情报告_{YYYY-MM-DD}.md`

## 工作流程

### Step 1：读取Excel文件，识别板块边界

```python
# 用openpyxl读取Excel，遍历所有行
# 通过识别板块标题行（"舆情"、"债圈热议"、"成交异动"、"一级发行"、"评级"、"公告"、"诉讼"）确定各板块的行范围
# 板块标题行特征：第一列有值，其余列均为空
```

### Step 2：舆情板块 — 筛选与分级

**排除规则**（与企业信用资质无关的信息，直接剔除）：


| 排除类型 | 二级/三级标签关键词    | 内容关键词                                              |
| ---- | ------------- | -------------------------------------------------- |
| 股价涨跌 | `股价上涨`、`股价下跌` | `涨停`、`涨幅`、`跌幅`、`股价`、`逆市走高`、`冲高`、`走高`、`板块拉升`、`板块短线` |
| 常规兑付 | `常规兑付`        | —                                                  |
| 无关新闻 | `其它新闻`        | `大宗交易`、`成交额超过`、`概念股`                               |
| 回购   | `股权变动`        | `回购`                                               |


**优先级分级规则**：


| 优先级     | 条件                                                     | 报告中标记                  |
| ------- | ------------------------------------------------------ | ---------------------- |
| 🔴 高风险  | 标记="风险" AND/OR (重要程度="重要" AND 标签含监管/财务异常/诉讼/评级变动/人事风险) | 重点展开描述                 |
| 🟡 重要关注 | 重要程度="重要"，或标签含评级动态/高管变动/减资/担保/持有人会议                    | 简要描述                   |
| ⚪ 一般信息  | 其余通过筛选的条目                                              | 不纳入报告（除非涉及主体数量极少时酌情补充） |


**重点关注的标签**（与信用资质强相关）：

- `行政处罚`、`监管处罚`、`监管问询关注` — 监管风险
- `财务异常` — 财务风险
- `诉讼纠纷` — 法律风险
- `评级上调`、`评级下调`、`评级动态` — 评级变动
- `人事风险`、`高层动态`（重要级别） — 治理风险
- `增持减持`（标记为风险的） — 股东行为异常
- `担保相关` — 或有负债风险
- `澄清回应`（重要级别） — 舆情应对

### Step 3：公告板块 — 筛选信用相关公告

**排除规则**（信息密度低、与信用资质无关）：

```
排除内容关键词：
同业存单、流通要素公告、发行情况公告、发行公告、上市公告、
付息公告、兑付公告、摘牌公告、兑付兑息、提示性公告、
可转债转股结果、可转债转股、债券上市流通、发行结果、
中签率、中签号、网上中签、网下发行、配售结果、回购
```

**保留规则**（与信用资质相关）：

```
保留标签/内容关键词：
信用评级、跟踪评级、担保、诉讼、仲裁、处罚、警示函、监管、
违约、逾期、减持、质押、冻结、业绩预告、业绩快报、
重大事项、重大资产、关联交易、对外投资、
债券持有人会议、减资、破产、重整、清算、立案
```

**分类整理**：

1. 跟踪评级报告 — 表格呈现
2. 新发债信用评级报告 — 表格呈现
3. 重大事项公告（高管变动/诉讼仲裁/减资/关联交易/持有人会议等） — 表格呈现

### Step 4：其他板块提取

- **债圈热议**：如有内容，直接纳入报告
- **成交异动**：提取估值偏离和前收涨跌数据，用表格呈现，附简要点评
- **一级发行**：提取发行/取消发行信息，重点关注取消发行的主体（可能反映再融资困难）
- **评级**：如有评级变动，重点标注
- **诉讼**：如有内容，纳入重点舆情

### Step 5：生成报告

## 关键原则

1. **信用视角优先**：所有筛选和判断都围绕"是否影响主体信用资质"展开，与信用无关的信息（股价涨跌、常规经营宣传、产品发布等）一律剔除
2. **风险信号前置**：🔴高风险事项必须排在最前面，确保读者第一时间看到
3. **同一主体合并**：同一主体的多条相关舆情应合并描述，避免重复
4. **舆情与公告交叉验证**：如舆情板块和公告板块出现同一事项（如诉讼、高管变动），应在报告中注明"舆情+公告双重披露"，增强可信度
5. **简洁高效**：报告目标是让信用研究员在5分钟内掌握当日所有需关注的信用事件，避免冗余信息

## 高频命令 /daily

当用户输入 `/daily` 时，按以下步骤自动执行：

### 自动化流水线（首选）

```bash
python daily_runner.py
```

该脚本串联以下五个步骤：

1. **`fetch_126_email.py`** — 从邮件拉取当日 DM 雷达日报，保存 HTML 正文
2. **`parse_email_html.py`** — 用 BeautifulSoup 解析 HTML，提取所有板块的结构化数据，输出 `raw_email_body_parsed.json`
3. **`generate_qige_report.py`** — 应用 CLAUDE.md 中的筛选规则（排除噪音、风险分级、同主体合并），生成 MD + DOCX 双格式报告
4. **`dm_daily.py`** — 从 DM 终端提取当日信用早报/债市要闻速览，生成格式化 DOCX
5. **`send_report_email.py`** — 通过 SMTP 发送邮件（HTML 正文 + DOCX 附件）

### 定时触发机制

| 触发源 | 时间 | 说明 |
|:---|:---|:---|
| **GitHub Actions cron** | 北京时间 07:30（工作日） | 主力定时器，UTC 23:30 Sun-Thu |
| **GAS 触发器** | 北京时间 09:00-10:00（工作日） | 兜底触发器，作为 DM早报补发机制 |
| **Windows 任务计划** | 北京时间 09:00（工作日） | 本地备用（需设环境变量） |

**去重机制**：
- GitHub Actions workflow 级别：cron 触发时检查当天是否已有成功/进行中的 run，有则跳过
- `daily_runner.py` 级别：北京时间 07:00-11:00 窗口限制 + 本地 sentinel 文件去重
- GAS 级别：dispatch 前检查当天是否已有成功 run，有则跳过
- workflow_dispatch（GAS/手动）不受 workflow 级别去重限制，确保 DM早报可补发

### 执行步骤

#### Step 1：拉取邮件

```bash
python fetch_gmail_dm.py
```

确认终端输出显示 `[完成]`。

#### Step 2：检查文件

确认以下文件存在且非空：
- `raw_email_body.html` — 邮件 HTML 正文（约 780KB）
- `raw_email_today.txt` — 文本摘要

如果文件不存在，告知用户"未拉取到当日邮件，请检查邮箱配置或网络连接"，终止流程。

#### Step 3：解析 HTML 提取结构化数据

```bash
python parse_email_html.py raw_email_body.html
```

输出 `raw_email_body_parsed.json`，包含：
- `yuqing` — 舆情条目列表（每条含 name / rating / tags / content / time / link）
- `gonggao` — 公告条目列表
- `chengjiao` — 成交异动数据（估值偏离 / 前收涨跌）
- `faxing` — 一级发行数据
- `pingji` — 评级变动数据

#### Step 4：生成报告

```bash
python generate_qige_report.py raw_email_body_parsed.json
```

应用以下规则：
1. **排除无关信息**：股价涨跌、常规兑付、大宗交易、概念股、回购等
2. **风险分级**：
   - 🔴 高风险：标签含行政处罚/监管处罚/监管问询关注/财务异常/诉讼纠纷/评级下调/人事风险
   - 🟡 重要关注：标签含评级动态/评级上调/高层动态/人事动态/增持减持/澄清回应/担保相关/减资/持有人会议/财报业绩
   - ⚪ 一般信息：其余（不纳入报告）
3. **公告筛选**：排除同业存单、发行/上市/付息/兑付/摘牌公告，保留跟踪评级、新发债信用评级、重大事项
4. **同主体合并**、**舆情+公告交叉验证**

输出文件（双格式）：
- `【大家资产持仓信用主体舆情日报】_YYYY-MM-DD.md` — Markdown 版本
- `【大家资产持仓信用主体舆情日报】_YYYY-MM-DD.docx` — Word 文档版本（专业排版、表格着色）

### 完成标志

终端打印报告生成路径，简要列出高风险主体数量和核心风险类型。

### 故障恢复

- **邮件拉取失败**：检查 `raw_email_body.html` 是否已存在，若存在可直接跳过 Step 1
- **HTML 解析失败**：检查 `raw_email_body.html` 是否完整（约 780KB），重新运行 `fetch_gmail_dm.py`
- **OSS Excel 链接 404**：时间限制签名 URL 过期属正常现象，HTML 解析为首选方案，不依赖 Excel 下载
